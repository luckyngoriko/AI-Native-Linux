## Maintainer Responsibilities

Maintainers are responsible for:

- keeping Rev.2 internally consistent
- reviewing security-sensitive authority flows
- preserving the typed action and policy-first execution model
- triaging issues and pull requests
- coordinating specification and future runtime implementation work
- keeping the public repository and project site aligned

## Review Areas

Security-sensitive areas include policy, sandboxing, vault access, package/service control, kernel bootstrap, filesystem authority, network exposure, verification, evidence, and recovery.

Changes in these areas should identify the affected layer and explain how the boundary is enforced.
